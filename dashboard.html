Come aggiornare il dashboard Workflow settimanale in 4 passi · click per aprire
⌄

1 Scarica i file da LinkedIn

LinkedIn Page Analytics → pagina RS Italia:

  • Tab Contenuti → Esporta → seleziona settimana → scarica XLS
  • Tab Follower → Esporta → scarica XLS

LinkedIn Campaign Manager:

  • Reports → Esporta performance → periodo: ultima settimana → CSV
Il nome dei file non importa: lo script li riconosce dal contenuto.

2 Metti i file in inbox/

Trascina qui tutti i file scaricati:

inbox/
  • 2 XLS LinkedIn (content + followers)
  • 1 CSV Campaign Manager
  • Qualsiasi nome va bene — lo script distingue i tipi da solo
Dopo l'update i file vengono spostati in archive/ automaticamente.

3 Lancia il comando unico

Apri il Terminale nella cartella del progetto:

python3 update.py --keye
  • Rileva e archivia i file da inbox/
  • Scarica lo snapshot Keye via API
  • Rigenera dashboard.html con tutti i dati aggiornati
↻
Copia il comando update Eseguilo nel terminale nella cartella del progetto

4 Ricarica il browser

Hard refresh per vedere i dati aggiornati:

Cmd + Shift + R

Come funzionano i filtri:

  • Settimanale: ultimi 7 giorni → confronto vs settimana precedente
  • Mensile / Storico: range più ampio, delta appare se hai storico sufficiente
  • Campagne: si filtrano per data come le altre tab
  • Audience e Brand Listening: sempre lo snapshot più recente
Struttura archivio: archive/linkedin/ · archive/keye/ · archive/ per CSV campagne · archive/pdfs/ per i PDF.
